# C# の全体像：プログラムの構造と基本概念

このドキュメントでは、Microsoft公式の「C# のツアー」の内容を、図解を交えて分かりやすく解説します。

---

## 1. C# プログラムの「3つの階層」
C# のコードは、整理整頓のために以下の3つの「箱」で構成されています。



### ① 名前空間 (namespace)
* **役割:** 関連するプログラムをまとめる「大きなフォルダ」です。
* **メリット:** 他の人が作ったプログラムと名前がぶつかるのを防ぎます。

### ② クラス (class)
* **役割:** プログラムを構成する「部品」です。
* **メリット:** 「データ（変数）」と「処理（メソッド）」をセットにして、一つの役割（例：`User`, `Order`）を持たせます。

### ③ メソッド (method)
* **役割:** 実際の「作業手順書」です。
* **メリット:** コンピュータはこの中に書かれた命令を、上から一行ずつ実行します。

## 2. 型 (Type) と変数：データの「種類」を厳守する
C# は **「型に厳しい（静的型付け）」** 言語です。変数を作る際は、必ず「何を入れる箱か」を指定します。

### 代表的な基本型
| 型名 | 内容 | 使う場面の例 |
| :--- | :--- | :--- |
| **int** | 整数 | 個数、年齢、ID |
| **double** | 小数 | 消費税、平均値、座標 |
| **string** | 文字列 | 名前、住所、メッセージ |
| **bool** | 真偽値 | `true`（はい） / `false`（いいえ） |

### 変数の宣言ルール
```csharp
int score = 100;           // 整数型の変数 score
string userName = "佐藤";   // 文字列型の変数 userName
```

## 3. 【最重要】値型 (Value Types) と 参照型 (Reference Types)

C#の学習において最も重要な概念です。データの持ち方には2つのパターンがあります。

### ① 値型 (Value Types)
* **イメージ**: 変数（箱）の中に、**データそのもの**が直接入っています。
* **対象**: `int`, `double`, `bool`, `struct`, `enum` など。
* **挙動**: 変数を別の変数にコピーすると、中身の数値がそのまま複製されます。コピー先を書き換えても、元の変数の値は変わりません。



### ② 参照型 (Reference Types)
* **イメージ**: 変数（箱）の中には、**「データ本体が置いてある場所（住所）」** が入っています。
* **対象**: `class`, `interface`, `delegate`, `object`, `string`（※特殊）, `array` (配列) など。
* **挙動**: 変数をコピーすると、**「住所」** が複製されます。そのため、2つの変数が「同じ一つの実体」を指すことになり、片方を書き換えるともう片方も変わって見えます。

## 4. プログラムが動く仕組み (コンパイル)

C# は人間が書いたコードをそのまま実行するのではなく、一度コンピュータが理解しやすい形に「翻訳」して動かします。

* **ソースコード (.cs)**: 私たちが Visual Studio で書くテキストファイル。
* **ビルド (Build)**: Visual Studio がソースコードを「中間言語 (IL)」という共通言語に翻訳する工程。
* **実行 (Runtime)**: プログラムを動かす際、.NET ランタイムがその場でOS（Windows等）に合わせた命令に変換して実行する。

---

## まとめ

1.  **階層構造**: プログラムは `namespace`（フォルダ） > `class`（部品） > `method`（手順） の順に整理されている。
2.  **型の厳守**: すべてのデータには「型」があり、違う型を混ぜることはできない。
3.  **参照の意識**: クラスなどの「参照型」を扱うときは、箱の中に「実体」ではなく「住所」が入っていることを意識する。
